How Giscus Reactions Work

Your portfolio uses GitHub-powered reactions for likes. This means all reactions are stored in GitHub and shared across all users - no database required!

What Users Will See

Below your blog articles, users will see a row of emoji reactions like this:

Interactive: Users can click any emoji to add their reaction. Click again to remove it. Reactions are instant and visible to everyone!

How It Works

  1. User Visits Blog Post
    Giscus automatically creates a GitHub Discussion for that page
  2. User Clicks Emoji
    They're prompted to sign in with GitHub (if not already signed in)
  3. Reaction Saved
    The reaction is stored in GitHub Discussions
  4. Visible to Everyone
    All visitors see the updated reaction counts in real-time

For You As the Owner

You can view and manage all reactions through GitHub:

  1. View Reactions
    Go to your repo ‚Üí Discussions tab ‚Üí See all reactions on each discussion
  2. Get Notifications
    GitHub can email you when someone reacts (configure in Settings)
  3. Export Data
    All reaction data is in your GitHub repo - export anytime
Pro Tip: You can see who reacted with what emoji by clicking on the reaction count in GitHub Discussions!

Setup Checklist

Before reactions work, make sure you've completed these steps:

  1. Enable GitHub Discussions
    Repository Settings ‚Üí Features ‚Üí Check "Discussions"
  2. Install Giscus App
    Visit github.com/apps/giscus and install
  3. Configure Giscus
    Go to giscus.app and get your config values
  4. Update Blog Files
    Replace placeholder values in your blog post HTML files
  5. Deploy & Test
    Push to GitHub, wait for deployment, test reactions!
